# REPLTA.MD

[English](README.md)

이 저장소는 에이전트가 프로젝트 안에서 어떻게 일해야 하는지를 정리한 작업 규칙을 담고 있다. 중심 문서는 `AGENTS.md`이며, `config.toml`은 에이전트 실행 환경의 기본 설정을 보조한다.

## 핵심 철학

`AGENTS.md`의 철학은 빠른 실행보다 통제 가능한 실행에 가깝다. 요구사항을 먼저 확인하고, 계획을 세운 뒤, 사용자의 승인을 받은 다음에 실제 변경을 수행한다. 이 흐름은 작업 속도를 늦추기 위한 절차가 아니라, 추측으로 인한 범위 확대와 사용자 변경사항 훼손을 줄이기 위한 안전장치다.

문서가 강조하는 원칙은 다음과 같다.

- 요구사항은 `REQ-*` ID로 정리해 무엇을 하기로 했는지 명확히 남긴다.
- 구현 방향은 `SPEC-*` ID로 계획해 요구사항과 실행 방식을 연결한다.
- 실행 작업은 `TASK-*` ID로 나누어 순서와 검증 기준을 드러낸다.
- 계획과 실행을 분리하고, 프로젝트 파일 변경 전에는 사용자 승인을 받는다.
- 저장소의 실제 상태를 먼저 조사해 불필요한 가정과 추측을 줄인다.
- 완료된 작업과 남은 작업을 분리해 현재 상태를 추적 가능하게 유지한다.
- 사용자나 다른 도구가 만든 변경사항을 되돌리거나 덮어쓰지 않는다.

## 작업 문서

`AGENTS.md`는 필요할 때 다음 작업 문서를 사용하도록 정의한다.

- `__REQ.md`: 요구사항 요약과 확인
- `__PLAN.md`: 요구사항에 연결된 실행 계획
- `__TASK.md`: 현재 진행 중인 작업 목록
- `__TASK.done.md`: 완료 및 검증된 작업 목록
- `__SPEEDWAGON.md`: 외부 검색으로 확인한 정보 기록

이 파일들은 작업 중인 노트로 취급된다. 필요한 만큼만 만들고, 짧고 현재 상태에 맞게 유지하는 것이 원칙이다.

## 저장소 구성

- `AGENTS.md`: 에이전트 작업 규칙과 승인 기반 워크플로우
- `config.toml`: 모델, 추론 강도, 성향, 기능 플래그 설정

## 운영 관점

이 저장소의 규칙은 에이전트를 독립 실행자가 아니라 협업자로 본다. 에이전트는 먼저 맥락을 읽고, 불확실성을 질문으로 남기며, 변경 전 승인을 받고, 실행 후 검증 결과를 공유해야 한다. 따라서 좋은 작업은 단순히 파일이 바뀐 상태가 아니라, 왜 바뀌었고 어떤 요구사항을 충족했는지 추적 가능한 상태다.
